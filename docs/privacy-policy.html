TrapGuard — Privacy Policy

Last updated: June 2026  ·  github.com/oribitan258/TrapGuard

What TrapGuard does

TrapGuard is a browser extension that detects hidden AI prompt-injection instructions in academic files (PDF, DOCX, PPTX, TXT, MD) before they are uploaded to AI assistants such as ChatGPT or Claude.

Data collection — none

TrapGuard collects no personal data and transmits nothing to any server. All file analysis happens locally inside your browser. File bytes never leave your device.

What stays on your device

Scan history is stored in your browser's local storage (via chrome.storage.local). Each history entry contains only:

  • The file name
  • The verdict (clean / infected / error)
  • The timestamp of the scan
  • The revealed hidden text (if a threat was found)

This data never leaves your browser. It is ring-buffered to approximately 100 entries and can be cleared at any time from the extension's settings page.

No third-party services

TrapGuard makes zero external network requests. It uses no analytics, tracking, crash reporting, or remote configuration. All fonts are self-hosted inside the extension package.

Permissions

The extension requests access to supported AI sites (chatgpt.com, claude.ai, and others you optionally add) solely to intercept file uploads at the moment of risk. It does not read page content, form data, or any information unrelated to file upload interception.

Open source

TrapGuard is open source. You can inspect the full source code at github.com/oribitan258/TrapGuard.
